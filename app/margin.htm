Организация отступов в верстке


Одним из основных принципом верстки является гибкость. А именно возможность легко вносить какие-либо изменения и ничего при этом не сломать.
Умение правильно обращаться с margin и padding играет очень большую роль в этом деле.

Основные принципы:
1. Отступы идут от предыдушего елемента к следуюшему.

2. Отступ задается последнему возможному елементу в доме.

3. Отступы нельзя задавать для независимых елементов ( бэм блок ).

4. У последнего елемента группы, отступ обнуляется(всегда).

 

Отступы идут от предыдушего елемента к следуюшему

margin(ы) задаются от предыдущего елемента к следующему, от первого ко второму, сверху вниз,  слева направо.

Это значит.что такие свойства как margin-left и margin-top не используются ( бывают исключения ). Если нужен отступ сверху, то используется padding-top для родителя. По тому же принципу  padding-left для родителя.

Зачем это нужно:
Важно соблюдать осмысленную структуру во всем, можно назвать это семантикой. Отступы идут в направлении потока дом дерева, блок сам себя не толкает.

 

Отступ задается последнему возможному елементу в доме

Это значит. что отступы задаюся соседним елементам дома. Для простого примера можно взять меню( список ссылок ).

ul>li*4>a

Чтоб сделать расстояние между ссылками задается отступ не для ссылки, а для лишки, лишка будет последним, и отступ будет между соседними елементами. Пример с меню довольно простой и банальный. Но если представить вложенность до 10 блоков, то можно намусорить маргинами и падингами.


Если взять body>h1, и нужно сделать отступ для заголовка сверху. то последний елементом будет body и для него задается padding-top, margin(ы)  которые стоят по дефолту всегда нужно обнулять.

 

Зачем это нужно:
Самая простая причина, по которой стоит придерживаться данного принципа, это чтоб было проще найти отступ в будущем, либо вам, либо тому, кто будет в дальнейшем работать с вашей версткой. Это если говорить про удобство.

Настоящие проблемы могут возникнуть, когда вывод блоков с плохой структурой отступов выводится динамически или дублируется.

 

Отступы нельзя задавать для независимых елементов ( бэм блок )

Никогда не делайте отступы елементов, которые могут использоваться больше чем один раз. Для этого есть обертки. Обертки это хорошо. Либо дополнительные классы.
Если нужно сделать блоку отступ.  Без ущерба это делается с помощью наследования.
.block__item > .block { margin-right: 10px; }

.user-description__avatar > .avatar { margin-right: 10px }
Таким же образом задается позиционирование, через наследование.

1. Наследование.
2. Добавление класса.
3. Обертка.

 

У последнего елемента группы, отступ обнуляется

Возьмем для примера список. Чаще чем другие псевдоклассы, вы должны использовать :last-child. Последний отступ никогда не нужен оставаться. Если есть расстояние между последней лишкой и новым блоком, то этот отступ делается не за счет лишки. А список отступает.

Так правильно:

Исключением обычно служит список товаров, где в ряду может быть от 2 ячеек и обнулять отрицательным маргином последние оступы не всегда целесообразно, но желательно.
